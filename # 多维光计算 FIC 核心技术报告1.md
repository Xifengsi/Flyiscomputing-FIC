## 一、MECU（存嵌计算单元）—— 物理层核心

### 1.1 定义
**MECU（Memory-Embedded Computing Unit，存嵌计算单元）** 是 FIC 架构中最小的、不可再分的物理执行核。它在一个约 8μm × 2μm 的微纳区域内，集成了光缓存、时序修正和全数字逻辑精算三大功能，所有操作在光域内随波包飞行瞬间完成。

MECU 是 FIC 架构的“地核”——整个芯片的确定性二进制数字计算与进位补码逻辑最终都在 MECU 中完成。

### 1.3 完整二进制运算覆盖

| 运算类型 | 物理实现机制 |
|---|---|
| **ADD（加法）** | MMI线性干涉XOR（求和） + 量子点饱和吸收超快偏转网（进位） |
| **SUB（减法）** | 前级路由物理换向至反相分量并注入末位+1（真二进制补码） → ADD |
| **MUL（乘法）** | 移位空间波导网络 + MMI数字加法树高速累加 |
| **CMP（比较）** | 减法器输出端无源零检测（明暗判决） |
| **AND（与）** | 饱和吸收区超快功率阈值路由输出 |
| **OR（或）** | 无源波导交叉合并汇聚点 |
| **NOT（非）** | 差分正反相轨道/分量直接物理交叉对调输出 |
| **XOR（异或）** | MMI线性干涉自成像格点空间路由 |
| **SHIFT（移位）**| 亚波长超表面空间交叉阵列 + pLatch动态时序控制 |

### 1.4 单核物理参数

| 参数 | 数值 |
|---|---|
| 总面积 / 几何尺寸 | ~0.0021 mm² / 长 8 μm × 宽 2 μm |
| 波长通道数 | 40 通道（C波段 DWDM，100GHz 间隔，40GHz 爆发频率） |
| 单次逻辑动作能耗 | < 5 fJ/bit （超低能耗数字光控光） |
| 静态/动态热设计功耗 | 计算核接近无源零发热（废热通过边缘吸光阱耗散） |

## 二、pLatch（光锁存器）—— 分布式光缓存

### 2.1 定义
**pLatch（photonic Latch，光锁存器）** 是基于硅基微环谐振器交叉耦合反馈的光学双稳态器件。它以光信号在闭环中的自持循环来保持状态，不依赖任何材料相变（如 PCM），不存在物理擦写磨损机制。

pLatch 是 FIC 架构的“光 SRAM”——每个 MECU 核自带的分布式、非共享私有缓存。

### 2.2 物理原理
- 两个紧凑型硅基微环谐振器（半径 $R=2\mu\text{m}$）通过非线性交叉耦合形成光学双稳态闭环。
- 写入波包触发状态翻转（SET/RESET），状态以光子循环形式自持保持。
- **排废机制**：状态切换时留在闭环内没飞干净的 5%~10% “尾巴光”（光子废水），不进入 MMI 计算腔，而是通过专用空间旁路排废通道直接导入外围的 **片上无源吸光阱**。

### 2.3 关键参数

| 参数 | 数值 |
|---|---|
| 读写与自持速度 | 20-40GHz |
| 擦写寿命 | >10¹⁵次（彻底解决 PCM 寿命墙） |
| 入口容量 / 出口容量 | 1-2KB（操作数对齐锁存） / 4-8KB（拥塞缓冲大坝） |
| 信号格式 | 与 CDOL 信号规范完全兼容（支持差分分量同步保持） |

## 三、LTC（本地时序自愈机构）—— 时序控制

### 3.1 定义
**LTC（Local Timing Corrector，本地时序自愈机构）** 是 MECU 核第 2 微米区域的物理层时序自适应控制部件。它在信号进入 MMI 计算腔之前的最后 2μm 内，在本地完成飞秒级的时序对齐，是 FIC 架构的“无源时序刹车片”。

### 3.2 工作原理
1. **检测**：前置无源比相器从差分分量中分出极微量光强，瞬时感应两路波前的时序偏差。
2. **修正**：功率差驱动异质键合的 **自组装铟镓砷（InAs/GaAs）量子点掺杂波导臂** 瞬时注入载流子，通过超快非线性等离子体色散效应踩下“物理刹车”，强行拉平飞秒级公差。
3. **恢复**：利用量子点特有的超快**俄歇（Auger）复合效应**，载流子在 **<300fs** 内完全释放并恢复基态，物理记忆彻底归零，不污染下一帧 40GHz 的紧跟波包。

### 3.3 关键参数与过渡方案

| 参数项目 | 量子点（InAs/GaAs）黑科技方案 | 工业级工程过渡替代方案 |
|---|---|---|
| **调节范围 / 恢复时间**| ±350fs / <300fs（超快原位复位） | 视器件带宽而定（>50GHz） |
| **过渡器件选择** | 前沿研究阶段物理底牌 | **电吸收调制器（EAM）** 或 **硅基MZM** |
| **协同关系** | 接收 pLatch 释放光流，为 MMI 输送绝对对齐的 CDOL 数字相位流 | 全程无全局电子时钟线，靠波包相对时差自愈 |

## 四、CDOL（同轨差分光逻辑）—— 信号规范

### 4.1 定义
**CDOL（Co-located Differential Optical Logic，同轨差分光逻辑）** 是 FIC 架构的基础信号定义规范。每个二进制状态由**同一条单模波导空间内、两个独立正交物理维度（如正交偏振态或正交空间模式）的差分相位**来绝对判定，而非由模糊的光强绝对值判定。

CDOL 是 FIC 架构的“0/1 判定法线”——整个芯片上流转的彩虹波包在二进制层面都死死遵循 CDOL 的硬核规则。

### 4.2 编码规则（离散二进制数字定标）

| 逻辑状态 | 正相偏振/模式分量（A） | 反相偏振/模式分量（¬A） | 数理与物理含义 |
|---|---|---|---|
| **逻辑 1 (True)** | 有光（相对参考相位为 0） | 无光 | 差分相对相位为 0（相长干涉） |
| **逻辑 0 (False)**| 无光 | 有光（相对参考相位为 $\pi$） | 差分相对相位为 $\pi$（相消干涉） |
| **无效/空状态** | 有光 / 无光 | 有光 / 无光 | 物理排废通道排除 / 无数据流 |

### 4.3 CDOL 同轨的共模抑制优势
- **传统双轨（分离波导）**：即使做到 5nm 间距，两条线的局部热、应力不均匀性依然存在，共模抑制比有上限，且存在严重的倏逝波串扰。
- **FIC 同轨（CDOL）**：正反相分量身处同一个微观空间格点，历经完全相同的温漂与流片公差。**环境扰动直接作为共模噪声在 MMI 干涉时被数学减掉（直接抵消）**，明暗对比度（消光比）刚性超越 20dB，具有绝对的确定性。
